\# GitHub Features Documentation



\## 1. Repositories

A repository stores project files and version history.



\## 2. Branching

Branches allow parallel development without affecting the main code.



\## 3. Pull Requests

Pull requests are used to propose changes and merge branches.



\## 4. Issues

Issues track bugs, improvements, and tasks.



\## 5. GitHub Actions

Used for CI/CD automation and workflow management.



\## 6. Releases

Releases help publish stable project versions.



\## 7. Insights

Shows contributor activity, commit history, and traffic.



\## 8. Forking

Allows copying a repository to your own account.



\## 9. Projects

Kanban-style task management.



\## 10. Discussions

Community interaction and Q\&A.
